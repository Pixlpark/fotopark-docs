* Данный раздел является основным при работе с сервисом. Проектом является класс или группа, для учеников которой создаются выпускные альбомы.
* Со стороны компании за проект отвечает менеджер, а со стороны класса - участник, которому предоставлен доступ к проекту через свой личный кабинет (ЛК). Обычно таким человеком выступает классный руководитель, староста или представитель родительского комитета, с которым заключается договор. Последний входит в ЛК через профиль своего ребенка. Для удобства дальнейшего описания функционала сервиса будем называть такого человека __координатором__ (их в проекте может быть несколько). Для назначения человека координатором надо в профиле ученика или педагога открыть доступ к проекту, а затем во вкладке “Настройки доступа” определить допустимые действия.
* Работа над проектом распадается на несколько этапов, переход между которыми осуществляется путем изменения его статуса. Рассмотрим их более подробно.
## Этап 1: Настройка проекта и загрузка фотографий
* После создания проекта ему присваивается статус “__Создан__”, а вся работа по его настройке осуществляется во вкладке “__Информация__”.
* В блоке “__Основная информация__” задаются:
    + Базовые характеристики, заполняемые при создании проекта.
    + Тип проекта, который определяет лишь доступный набор цитат для выбора (если это подразумевает дизайн). Т.е. для выбора будут отображаться цитаты с таким же типом.
    + Количество учеников для автоподстройки дизайна (изменения набора разворотов с виньетками). По умолчанию значение поля совпадает с количеством созданных профилей учеников. Но его можно переопределить на произвольное. Например, при создании проекта указать количество учеников, если известно.
* В блоке “__Дополнительная информация__” задаются:
    + Профиль класса, если это подразумевает дизайн.
    + Персонализация альбома оживающими фотографиями с загрузкой иллюстрации QR-кода.
    + Автоматический отбор снимков для ученика, если они не выбраны или их недостающее количество. Вначале идет отбор из общих фотографий проекта, а затем среди всех фотографий, на которых отмечен ученик.
* В блоке “__Дедлайны__” задаются крайние сроки для подписи и выбора фотографий, проверки дизайна, обработки фото и утверждения альбомов. Для каждого события предусмотрены уведомления менеджера по заказу, профильного сотрудника, координатора и учеников.
* В блоке “__Фотосессии__” отображаются запланированные и проведенные фотосессии. После создания фотосессии предусмотрено уведомление фотографа, на которого она назначается.
* В блоке “__Дизайны__” определяются пресеты, которые в рамках проекта могут быть перенастроены. Если дизайнов несколько, то в профиле ученика можно выбрать любой из них. По умолчанию будет использован первый. Для каждого дизайна также выбирается мастер-альбом. Именно в нем редактируются общие развороты, виньетки с учениками и педагогами.
* В блоке “__Настройка доступа__” задаются:
    + Возможность выбора цитаты из готовых, а не только написание собственной.
    + Редактирование в альбоме разворотов, где присутствуют репортажные снимки. При включении дополнительной настройки разрешается редактировать абсолютно все развороты без ограничений.
    + Выбор персональных репортажей вместо использования общих для всех учеников.
    + Скачивание всех снимков фотосессий из личного кабинета. По умолчанию скачивание запрещено, а на все фото накладываются водяные знаки.
    + Объединение выбора фотографий для виньеток, персональных фото и на обложку. Можно выбрать любую комбинацию для объединения.
